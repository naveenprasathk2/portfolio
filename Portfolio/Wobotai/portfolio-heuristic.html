Heuristic Evaluation of VisionGuard

01 — Project overview

Why this evaluation matters

VisionGuard is an AI-powered video analytics platform that helps businesses monitor operations through camera feeds. As a complex B2B SaaS product with dashboards, analytics, and real-time monitoring, it presented a rich surface area for usability analysis.

Product analysed
VisionGuard — an AI video analytics platform used for operational monitoring, compliance, and workplace safety
Evaluation scope
Full product interface including navigation, dashboard, settings, icons, layout, help documentation, and error handling
Goal
Identify usability issues that could cause user confusion, task failure, or inefficiency — and propose evidence-based improvements
Deliverable
Structured evaluation matrix with severity ratings, specific findings per heuristic, and actionable design recommendations
02 — Methodology

A structured approach to expert review

I applied Jakob Nielsen's 10 Usability Heuristics as a systematic lens to evaluate each area of the VisionGuard interface, documenting findings using a question → assessment → recommendation structure.

1

Define scope

Identified core user tasks and interface areas: navigation, dashboard cards, settings, icons, content hierarchy, help pages, and error states.

2

Apply heuristics

Systematically walked through each of Nielsen's 10 heuristics, formulating evaluation questions specific to the VisionGuard interface.

3

Document findings

Recorded each finding with a yes/no assessment, specific examples from the interface, and the design principle being violated.

4

Rate severity

Assigned severity levels (high / medium / low) based on frequency, impact on task completion, and persistence of the issue.

5

Recommend fixes

Proposed specific, actionable design improvements tied to each finding — not vague suggestions, but concrete UI changes.

03 — Key findings

What the evaluation uncovered

Across all 10 heuristics, I identified 20+ usability issues. Here are the most significant findings grouped by heuristic.

1
Visibility of system status
Medium
Question
Is the terminology of the menu items user-friendly? Does the screen start with a title that describes the page content?
Finding
Menu terminology is not fully user-friendly — terms like "Payouts" and "Expense" may confuse users. Icon choices need improvement: Insights uses a graph icon, Settings and Logout use non-standard icons.
Recommendation
Replace non-intuitive terms with plain language. Use standard, universally recognized icons. Add icon captions for new or unfamiliar icons.
2
Match between system and real world
Medium
Question
Is the system designed with users' habits in mind? Does the system speak users' language with familiar words rather than technical terms?
Finding
Navigation is somewhat familiar but placed at top/sidebar only. Complex words like "Commerce," "Payouts," and "Pledge" could be replaced with simpler terms. The plus icon next to the notification icon doesn't indicate its purpose.
Recommendation
Replace complex words with familiar, simple terms. Provide a relevant icon or label next to the plus icon. Enable drag-and-drop for card rearrangement.
3
User control and freedom
High
Question
Can the user change the layout as per their requirement? Is dark/light theme option available? Can users save their layout?
Finding
Users cannot customise their layout or rearrange dashboard cards. There is no dark/light theme toggle despite the product being used for extended monitoring sessions. Users cannot save layout preferences.
Recommendation
Add a toggle button or switch for dark/light theme as part of accessibility. Allow users to drag, sort, and save card arrangements per their needs.
4
Consistency and standards
Medium
Question
Are the main navigation items always available and not hidden? Are the icons intuitive and consistent?
Finding
Main navigation is always visible, which is good. However, sub-items could be clearer. The logout icon uses a non-standard visual — a caption would make it more meaningful. Some icons lack consistency across pages.
Recommendation
Add captions to non-standard icons. Use the same icon style (filled vs outlined) consistently. Ensure the logout icon follows platform conventions.
5
Error prevention
Low
Question
Since there isn't any forms or input fields, can error prevention checks be done elsewhere?
Finding
The product currently has limited form inputs, reducing the surface area for input errors. However, the dashboard lacks data validation or confirmation patterns for any user-initiated actions.
Recommendation
Add confirmation dialogs for destructive actions. Implement inline validation if forms are introduced in future iterations.
6
Recognition rather than recall
Medium
Question
Do the system or product use different words, actions, or situations to derive the same meaning? Does component placement follow familiar mental models?
Finding
The product does not confuse users with different terms for the same meaning — this is acceptable. However, some components like the search bar could be elaborated and kept as a visible input field for easy access rather than hidden behind an icon.
Recommendation
Keep the search bar as a persistent input field rather than collapsed behind an icon. Follow users' mental models for component placement (e.g., cart icon, search bar position).
7
Flexibility and efficiency of use
Medium
Question
Can experienced users take advantage of accelerators and other secondary features designed to speed up commonly performed actions?
Finding
The product offers some accelerators for experienced users, but power-user features are limited. There are no keyboard shortcuts, bulk actions, or customisable workflows visible in the interface.
Recommendation
Introduce keyboard shortcuts for frequent actions. Add bulk selection and batch operations. Consider customisable dashboard widgets for power users.
8
Aesthetic and minimalist design
Medium
Question
Is the information essential to decision-making displayed on the screen? Are universal visual patterns used consistently?
Finding
Essential information is displayed, but the visual hierarchy could be stronger. Some areas need more visual connection to make the design better and provide a good user experience. The outlined Insights icon could be replaced with a filled icon to differentiate from others.
Recommendation
Strengthen visual hierarchy through spacing and contrast. Use filled vs outlined icons consistently to indicate active states. Reduce visual clutter in dense data areas.
9
Help users recognise, diagnose, and recover from errors
High
Question
Is the FAQ page user-friendly — divided into categories and searchable? Do important explanations remain displayed on the screen?
Finding
The FAQ page is missing entirely. Important explanations should remain on screen as long as the user needs them — the user should not have to write down explanations anywhere. No contextual help is available through icons.
Recommendation
Build a searchable, categorised FAQ section. Add contextual help through tooltip icons. Ensure error messages persist on screen and provide clear recovery steps.
10
Help and documentation
High
Question
Is there a live chat on every page of the site or application? Is contextual help available through icons?
Finding
Live chat is missing entirely — there is no way to get real-time help. Few info icons are available as contextual help through the interface. The product lacks onboarding guidance for new users.
Recommendation
Add a persistent live chat widget or help button on every page. Implement contextual tooltips and info icons throughout. Build an onboarding flow for first-time users.

05 — Reflection

What this project demonstrates

Systematic UX thinking

This evaluation demonstrates the ability to apply established UX frameworks methodically — not just identifying issues, but structuring findings in a way that's actionable for design and development teams.

Evidence-based recommendations

Every recommendation is tied to a specific finding and heuristic principle. This approach aligns with human factors methodology: grounding design decisions in established usability evidence rather than personal preference.

Human factors relevance

Heuristic evaluation is a core method in IEC 62366-1 formative usability evaluation for medical devices. This project demonstrates my ability to conduct systematic expert reviews — a skill directly transferable to MedTech human factors engineering.

What I'd add next time

In a professional context, I would complement this expert review with user testing sessions to validate findings, add task-based severity scoring, and create a prioritised remediation roadmap with effort estimates.
